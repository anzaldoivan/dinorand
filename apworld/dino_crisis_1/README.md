# Dino Crisis 1 — Archipelago world

A Python [Archipelago](https://archipelago.gg) world that lets AP **generate** a logically-valid
Dino Crisis 1 seed from DinoRand's authored logic. The **runtime half ships too** (contract v2):
slot_data carries AP's item placement (`placements` + `item_ids`), and the DinoRand CLI's
`--ap-connect` client patches it into the local GOG install and syncs the running game
(pickups → checks, received items → inventory, goal → completion) — live-verified end-to-end
against AP 0.6.7. See `docs/decisions/cross/AP-CLIENT-PLAN.md` and the USER-GUIDE "Archipelago"
section. Play-side requirements: Windows host (the client reads `DINO.exe` memory), server ≥ AP 0.6.x.

Locations flagged `excluded` in the contract share an in-game taken-flag with a twin pickup
(script-choreographed spots) — AP keeps progression items out of them automatically.

## Layout
| File | Role |
|------|------|
| `world.py` | `DinoCrisis1World` — regions / locations / items / rules |
| `data.py` | loads `data/dc1_logic.json`, builds name→id maps |
| `options.py` | `DinoCrisis1Options` (empty stub — required by AP) |
| `data/dc1_logic.json` | the contract, generated by `scripts/gen_ap_logic.py` (do not hand-edit) |
| `archipelago.json` | AP world manifest |
| `test/` | AP-side integration tests (`WorldTestBase`) — run inside an AP checkout |
| `selfcheck.py` | AP-independent structural check — runs anywhere, no AP install |

## Install as a packaged .apworld (official AP installer)
Zip this folder so the archive contains `dino_crisis_1/` at its top level, rename it
`dino_crisis_1.apworld`, and drop it into the AP install's `custom_worlds/`. Generate with a
player YAML as usual (`game: Dino Crisis 1`).

## Generate a seed with an Archipelago checkout
```bash
# 1. Get Archipelago and its deps
git clone https://github.com/ArchipelagoMW/Archipelago
cd Archipelago
python ModuleUpdate.py          # installs required Python packages

# 2. Drop this world in (copy or symlink the package folder)
cp -r /path/to/dinorand/apworld/dino_crisis_1 worlds/
#   ln -s /path/to/dinorand/apworld/dino_crisis_1 worlds/dino_crisis_1   # (symlink alt.)

# 3. Add a minimal player YAML
cat > Players/DC1.yaml <<'YAML'
name: DC1Player
game: Dino Crisis 1
Dino Crisis 1: {}
YAML

# 4. Generate
python Generate.py --seed 1
```
Success = `output/AP_*.zip` is written with no `FillError`/logic exception. Unzip it and read the
spoiler to confirm the three key items (BG Area Key, C. O. Area Key, Key Card Lv. A) are placed in
reachable spots and the Underground Heliport (060d) checks sit behind Key Card Lv. A.

## Tests
**AP-side integration** (run from the Archipelago root, after step 2 above):
```bash
python -m pytest worlds/dino_crisis_1/test          # this world's tests
python test.py                                       # AP's full suite incl. generic world tests
```
`WorldTestBase` also enrolls this world in AP's generic tests (`test_fill`,
`test_all_state_can_reach_everything`, `test_empty_state_can_reach_something`, id-uniqueness).

**AP-independent** (run from the DinoRand repo — no AP install needed, safe for CI):
```bash
python3 scripts/gen_ap_logic.py --check      # contract + reachability + gate-bites + forward-fill
python3 apworld/dino_crisis_1/selfcheck.py   # world.py structural wiring via a minimal AP stub
```

## Regenerating the logic contract
`data/dc1_logic.json` is distilled from the authored `data/dc1/{map,items,room-data}.json`:
```bash
python3 scripts/gen_ap_logic.py --apply   # regenerate + stage the result
```
Re-run after any `data/dc1` change. The pre-commit hook and the `apworld-checks` CI workflow run
`--check` (non-mutating) to reject a stale committed contract.
